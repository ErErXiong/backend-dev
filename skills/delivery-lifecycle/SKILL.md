# 完整研发交付

适用于用户明确要求全流程，或任务包含新模块、跨服务、迁移、权限或发布风险。

1. 先确认目标、非目标、使用者、验收标准、数据影响和发布约束；没有足够信息时先做方案，不实现。
2. 给出可审查方案：接口与数据契约、组件边界、失败处理、迁移与回滚、测试和上线验证。存在明显替代方案时使用 `brainstorming`；高风险决策使用 `grill-me`。
3. 用户确认方案后分阶段实现。每阶段运行相应测试，并进行独立代码审查，重点检查权限、输入校验、事务、兼容性、日志和敏感信息。
4. 有前端或浏览器可测路径时调用 `backend-dev:browser-qa`，核对真实请求、错误提示和刷新结果；后端改动不能替代前端验证。
5. 发布前列出配置差异、数据库步骤、回滚、监控指标和线上验收项。发布、推送和生产操作都必须先得到用户确认。
6. 线上验证后转入 `backend-retro`，只生成有可复用价值的候选经验。
