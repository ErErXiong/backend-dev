# 后端经验复盘

只在有可靠结论时执行。可运行 `scripts/collect-retro-context.py <仓库路径>` 收集只读 Git 上下文；不要把原始对话、Token、密码、内部地址或敏感数据写入候选。

不触发场景：纯问答、只读分析、无文件改动、纯格式修改，或任务失败但没有可靠结论。

候选最多 5 条。每条使用以下格式：

```text
标题：
发生了什么：
根因或规律：
下次如何避免：
检测信号：
置信度：高 / 中 / 低
建议位置：pitfalls.md / decisions.md / prompts.md / CONTEXT.md
```

先展示候选并等待用户回复 `写入`、`只写第 N 条`、`不写` 或修改文本。不得静默写入。确认后按项目名写入当前仓库的 `ai-memory/<项目名>/`：

- 根因、踩坑和检测信号：`pitfalls.md`
- 架构或业务决策：`decisions.md`
- 可复用步骤：`prompts.md`
- 领域术语：`CONTEXT.md`

修改 `AGENTS.md`、创建 ADR、提交、推送或发布必须分别再次确认。
