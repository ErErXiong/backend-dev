# 日常后端开发

按以下顺序工作：

1. 查看当前仓库状态、相关代码与现有测试。存在未提交改动时，先说明并避开无关文件。
2. 仅在影响范围、契约或验收标准不清时提出一个关键问题；否则给出简短计划，列明修改点和验证方式。
3. 以最小改动实现，保持既有接口、异常处理和项目风格；不要顺带重构无关代码。
4. 运行与变更匹配的本地验证。接口、Excel 和前后端契约问题要验证字段名、请求体、导入/导出列及调用链；有浏览器可验收路径时必须调用 `backend-dev:browser-qa`，或说明无法浏览器验证的具体原因。
5. 汇报验证结果和未验证的原因。检查 `git diff`，只建议暂存本次文件；提供提交信息建议。
6. 未经用户明确确认，不执行 commit、push、发布、数据库写入或破坏性命令。
7. 如果本次完成了实质性代码改动且得到可靠结论，转入 `backend-retro`；纯问答、只读分析、纯格式调整或没有可靠结论时静默结束。
